# Workflow: Validate PRD

**Agent:** Product Manager
**Shortcut:** `*validate-prd` or `VP`
**Phase:** 2 - Planning

## Purpose

Validate the PRD for completeness, consistency, and feasibility before proceeding to architecture.

## Validation Checklist

### 1. Completeness
- [ ] All required sections present
- [ ] User personas defined
- [ ] Success metrics specified
- [ ] Scope clearly defined
- [ ] Dependencies listed

### 2. User Stories Quality
- [ ] Stories follow "As a... I want... So that..." format
- [ ] Acceptance criteria defined for each story
- [ ] Stories are independent and valuable
- [ ] Stories are appropriately sized

### 3. Requirements Clarity
- [ ] No ambiguous language
- [ ] Technical terms defined
- [ ] Measurable where possible
- [ ] No conflicting requirements

### 4. Feasibility Assessment
- [ ] Technical feasibility confirmed
- [ ] Dependencies are available/accessible
- [ ] Scope matches available resources
- [ ] Risks are acknowledged

### 5. Alignment Check
- [ ] Aligns with project brief
- [ ] Addresses identified user pain points
- [ ] Differentiates from existing solutions
- [ ] Supports success criteria

## Validation Report

Generate a validation report with:
- Pass/Fail status for each check
- Issues requiring attention
- Recommendations for improvement
- Overall readiness assessment

## Outputs
- PRD validation report
- Updated PRD (if changes made)

## Next Steps
If validation passes:
- Proceed to `*create-architecture` with Architect agent

If validation fails:
- Address issues and re-validate
