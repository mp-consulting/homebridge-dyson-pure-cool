# Workflow: Sprint Retrospective

**Agent:** Scrum Master
**Shortcut:** `*retrospective` or `RT`
**Phase:** 4 - Implementation

## Purpose

Conduct a sprint retrospective to reflect on what worked, what didn't, and how to improve.

## Retrospective Format

### 1. Review Sprint Metrics

| Metric | Target | Actual |
|--------|--------|--------|
| Stories Completed | | |
| Story Points | | |
| Bugs Introduced | | |
| Bugs Fixed | | |

### 2. What Went Well
Celebrate successes and positive patterns:
- [Positive outcome 1]
- [Positive outcome 2]
- [Positive outcome 3]

### 3. What Could Be Improved
Identify areas for improvement:
- [Improvement area 1]
- [Improvement area 2]
- [Improvement area 3]

### 4. Action Items
Concrete improvements for next sprint:

| Action | Owner | Due |
|--------|-------|-----|
| [Specific action] | [Who] | [When] |

### 5. Shoutouts
Acknowledge contributions:
- [Recognition]

## Retrospective Questions

### Process
- Did our planning accurately predict the work?
- Were stories properly sized?
- Did we have clear acceptance criteria?

### Technical
- Is our code quality improving?
- Are tests catching issues?
- Is our architecture holding up?

### Communication
- Were blockers identified quickly?
- Did we have enough context for stories?
- Was the documentation helpful?
